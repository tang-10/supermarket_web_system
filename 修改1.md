### 二、 后端修改

#### 1. 路由层：统一 WebSocket 推送 (`src/api/routers/recognition.py`)
我们需要在 WebSocket 循环中读取视频、推理、编码图片为 Base64，并打包 JSON 发送。

```python
import cv2
import base64
import asyncio
from fastapi import APIRouter, WebSocket, WebSocketDisconnect
from src.api.dependencies import AppContext
from src.pipelines.recognition import RealtimeRecognitionPipeline
from src.api.ws_manager import ws_manager

router = APIRouter()

@router.websocket("/ws/recognition")
async def websocket_endpoint(websocket: WebSocket, video: str = '0'):
    await ws_manager.connect(websocket)
    
    # 1. 确定视频源
    if not video.isdigit():
        from pathlib import Path
        source = str(Path("data/test_videos") / video)
    else:
        source = int(video)

    pipeline = RealtimeRecognitionPipeline(
        AppContext.model_mgr, AppContext.vector_db, AppContext.product_db
    )
    cap = cv2.VideoCapture(source)
    frame_counter = 0

    try:
        while cap.isOpened():
            ret, frame = cap.read()
            if not ret:
                cap.set(cv2.CAP_PROP_POS_FRAMES, 0)
                continue

            # 2. 推理获取结果
            # 注意：process_frame 需返回数据列表，不画框
            results = pipeline.process_frame(frame)
            frame_counter += 1

            # 3. 每 2 帧推送一次数据（降频优化）
            if frame_counter % 2 == 0:
                # 编码图像
                _, buffer = cv2.imencode('.jpg', frame, [cv2.IMWRITE_JPEG_QUALITY, 70])
                img_base64 = base64.b64encode(buffer).decode('utf-8')

                # 构造发送数据
                # 确保 RecognizeResult.to_dict() 包含 price (从 DB 获取)
                data_to_send = [res.to_dict() for res in results]

                await websocket.send_json({
                    "image": img_base64,
                    "results": data_to_send
                })

            await asyncio.sleep(0.03) # 维持约 30fps

    except WebSocketDisconnect:
        ws_manager.disconnect(websocket)
    finally:
        cap.release()
```

### 三、 前端修改

#### 1. Pinia 状态更新 (`src/store/cart.js`)
修改结算逻辑：不再是累加历史，而是**实时反映当前帧**看到的物体数量。

```javascript
export const useCartStore = defineStore('cart', () => {
  const currentFrameItems = ref([]); // 存当前帧的商品

  const totalPrice = computed(() => {
    return currentFrameItems.value.reduce((sum, item) => sum + item.price * item.count, 0);
  });

  // 更新当前帧商品：根据数据进行分组计数
  function updateFromFrame(results) {
    const groups = {};
    results.forEach(res => {
      if (res.fine_class === 'unknown') return;
      if (!groups[res.sku]) {
        groups[res.sku] = { 
          sku: res.sku, 
          name: res.product_name, 
          price: res.price, 
          count: 0 
        };
      }
      groups[res.sku].count++;
    });
    currentFrameItems.value = Object.values(groups);
  }

  function clearCart() {
    currentFrameItems.value = [];
  }

  return { currentFrameItems, totalPrice, updateFromFrame, clearCart };
});
```

### 关键改进说明：

1.  **架构极致统一**：后端 `generate_video_stream` 现在是一个纯粹的消息生产者，它把图像和坐标封装在一个 JSON 里发出去。前端 Canvas 成了唯一的渲染出口。
2.  **结算逻辑精准化**：通过 `updateFromFrame` 函数，结算清单每秒都在根据最新的画面进行“重写”。这意味着如果一个苹果离开了画面，它会立刻从清单中消失；如果是两个苹果，数量会显示为 2。这符合商超实时核验的逻辑。
3.  **UI 细节优化**：
    *   **表格化清单**：增加了表头和清晰的行列划分。
    *   **清空逻辑**：点击“停止侦测并清空”会断开 WS 连接、重置文件选择状态、重置 Pinia 状态、清空画布。
    *   **无感背景加载**：使用 `new Image()` 的 `onload` 事件驱动 Canvas 绘制，有效防止了图像闪烁和由于图片未加载完成导致的黑屏。
4.  **性能**：通过 `IMWRITE_JPEG_QUALITY` 降质（70）和后端降频（每 2 帧发一次），在保持流畅度的同时，带宽压力大幅降低。
